## Digital Business Card

A fifth program type where users create a digital business card replicating a real business card, shareable via QR code or link. Recipients add it to their wallet (Apple/Google) as a contact card.

**Why it could work:**
- Reuses existing wallet pass infrastructure (Apple/Google)
- Reuses QR code generation and sharing flow
- Restaurant owners already have branding assets in the system (logo, colors)
- Natural upsell for paid plans

**Why to wait:**
- Different use case (networking vs. loyalty/retention) — risks diluting product focus
- Business card passes use different pass types (Generic pass, not Loyalty)
- No visit/reward/enrollment cycle — needs its own simplified flow
- Competitive space (Linktree, HiHello, Popl, etc.)

**If built, key features:**
- Card editor reusing the studio (name, title, company, phone, email, website, socials)
- QR code + shareable link
- Apple Wallet Generic pass / Google Wallet Generic pass
- vCard (.vcf) download fallback
- Analytics (how many times shared/viewed)

**Estimated scope:** New program type or standalone feature, ~1 phase
